PostGres Database

Benefits of using Postgres

Plus we can practive some SQL with it which will be a great skill to have in the future that complements fronted development skills really well.

Kinds of Databases

  1. Relational (SQL: Postgres, MySQL, SQLite)
  2. NoSQL
    • Document (JSON: Firebase, MongoDB)
    • Graph (Neo4j, ArangoDB)
    • Columnar (RedShift, Cassandra, ClickHouse)
    • Key-Value Store (DynamoDB, Couchbase, Redis)
    • Time Series (Prometheus, InfluxDB, TimescaleDB)

Using Bulma

We'll globally import Bulma CSS framework to get nice CSS automatically.

Just add this import to the top of your global index.css file:

@import "https://cdn.jsdelivr.net/npm/bulma@0.9.4/css/bulma.min.css";

API tokens & secrets

Don't push your secrets or tokens to Github

Create a .env file in your project root & then .gitignore those files so they stay only on your local computer

Import the secrets/tokens into your project securely (usually with dotenv pkg) or for Vite.js using import.meta.env.VITE_SECRET_NAME

Deploying Vite to Vercel Hosting

  1. Why can't we deploy our CRUD app to github pages? Because of the env vars
    2. In your Vite project:
    • Update your Vite project's vite.config.js to the default configuration. It should look like this: 
      import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
})
    • Add a file called vercel.json to the root of your project directort. It should have this content inside: 
      {
  "rewrites": [
    { 
      "source": "/(.*)", 
      "destination": "/" 
    }
  ]
}
    • Push these changes up to your remote project repo in Github.com
    3. At Supabase.com:
    • In the Supabse console, go to Authentication > URL in the sidebar & change the site url to the url given in the Vercel project console.
    • Add https://**vercel.app/*/* to the Redirect URLs panel
    4. At Vercel.com
    • Create an account in Vercel and start a new project. Link it to your Github project repo.
    • Add your environment variables (VITE_SITE_URL, VITE_ANON_KEY) to the Settings > Environment Variables section in Vercel
    • Click Deploy Site in your Vercel Dashboard
