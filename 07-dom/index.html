THE DOM API vs JS

The DOM API Looks like JS, but it's separate. Yet it interfaces with JS.

DOM represents the built-in objects, methods & properties that interface w/ nodes in the Render Tree

JS is the language that can store, iterate-over and manipulate those nodes

The link between JS & the DOM is key to building dynamic apps as it allows JavaScript to change your HTML reactively based on user input triggers.

Listening to Events

document.addEventListener('submit', callback)

Then we need to handle that event with our callback function that we define

For Example:

button.addEventListener('click', function(e) {
  console.log(e);
  const div = document.createElement("div");
  const text = document.createTextNode(
    "You clicked @ " + new Date().toLocaleTimeString()
  );
  div.appendChild(text);
  e.target.insertAdjacentElement("afterend", div);
})

Visualizing Events

DomEvents.dev

When you click a dom element, the browser walks the DOM tree from the HTML root down to the clicked element capturing intermediate elments along the way

When the traversal reaches the clicked element, it fires the event handler and then bubbles back up the tree to the HTML root firing all other event handlers it encounters along the path it previously traversed.

If you want you can set the optional {capture: true} to have it trigger handlers on the traversal downwards, instead of upwards.

Wes Bos on Event Capturing & Bubbling

Event Listeners Options

  • { capture: true }
  • { once: true }
  • e.stopPropagation()
div.addEventListener('click', e => {
    e.stopPropagation() // stops bubbling at this element
    // handle event
  },
  {
    capture: true, // fires handlers on traversal down
    once: true // only runs handler once
  }
)}
