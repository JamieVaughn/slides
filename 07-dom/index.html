DOM vs JS

THE DOM API vs JS

  • The DOM API Looks like JS & it interfaces with JS.
  • But the DOM API & JS are separate, distinct things.
  • DOM API = browser's built-in properties, methods & objects
  • Javascript = language for iterating, transforming & storing data
  • Javascript can change HTML reactively using the DOM API based on user input triggers. Using JS & the DOM API together is key to creating dynamic apps.

List of DOM Events

Three event types we will work with to start off:

  • click
  • keydown, keyup
  • submit

Add an Event Listener

document.addEventListener('submit', callback)

Then we need to handle that event with our callback function that we define

For Example:

button.addEventListener('click', function(e) {
  console.log(e);
  const div = document.createElement("div");
  const text = document.createTextNode(
    "You clicked @ " + new Date().toLocaleTimeString()
  );
  div.appendChild(text);
  e.target.insertAdjacentElement("afterend", div);
})

Listening to Events in HTML

Using the onclick attribute as an alternative way to listen to events

<!-- HTML file -->
<body>
  <!-- Add `onclick` attr to button
  & invoke your function -->
  <button onclick="my.func()">
    Click Me
  </button>
</body>
// .js file
const my = {
  func: () => console.log('Clicked!'),
}
// Or
const btn = document.querySelector('button')
// assign function to the `onclick` attr:
btn.onclick = my.func

Visualizing Events

DomEvents.dev

When you click a dom element, the browser walks the DOM tree from the HTML root down to the clicked element capturing intermediate elments along the way

When the traversal reaches the clicked element, it fires the event handler and then bubbles back up the tree to the HTML root firing all other event handlers it encounters along the path it previously traversed.

If you want you can set the optional { capture: true } to have it trigger handlers on the traversal downwards, instead of upwards.

Wes Bos on Event Capturing & Bubbling

Event Listeners Options MDN

  • { capture: true }
  • { once: true }
  • e.preventDefault()
  • e.stopPropagation()
document.addEventListener('click', e => {
    e.preventDefault() // prevents default event behavior
    e.stopPropagation() // stops bubbling at this element
    // handle event
  },
  {
    capture: true, // fires handlers on traversal down
    once: true // only runs handler once
  }
)}
