Hypertext Mark-up Language

Hypertext Mark-up Language (HTML)

  • Structures the document and tells browsers what a certain element's function is
  • Content is "marked-up" using tags
  • Tags usually (but not always) come in pairs,
    <p>This is an example of a paragraph tag or element</p>
  • What's a self-closing tag? 
    <br/> <img/> <input/>
  • The opening tag, closing tag and internal content generates an HTML element
  • Tag vs. Element?
    • Tag: the code as written
    • Element: the code object in computer memory

Tags are markers wrapped around text in the document to indicate how user-agents should interpret it

History of HTML

  • Invented by Tim Berners-Lee in 1989 as a means for easy linking of references in research papers using hypertext
  • Created the Hypertext Transfer Protocol (HTTP) for retrieving text from other documents via hypertext links
  • Specifications were introduced as a guide for standardisation among browser vendors
  • 1994: the URL spec was defined. protocol://host.domain:port/path/subpath?query#fragment

Document Type

<!DOCTYPE html>
  • Appears just above the opening <html> tag
  • Tells the browser to render the HTML in standards mode
  • Let's validation software know which version of HTML to validate against
  • Advised to use the HTML5 doctype

Standards mode means browsers try to render the page according to CSS specifications
Outdated or incomplete doctypes will be rendered in quirks mode, things may not appear as you expect

<html> element

<html lang="en">
	<!-- HTML code for web page -->
</html>
  • Represents the root of an HTML document
  • Encouraged to specify a language attribute
  • Language attribute aids speech synthesis (screen readers), translation tools and other language-related functionality

Semantics: Buttons vs Links

When to use <button> and when to use <a>

  • Buttons are for executing some javascript code
  • Links are for navigating to a new view (usually a new page URL)
  • Navigating to another part of the current page? Go To Previous Slide

Neilson User Research

Image Semantics:

Visual Content or Visual Decoration

<img src="..."> vs background-image: url(...)

  • Use the <img> tag if the image represents actual page content
  • The <img> tag is accessible & can be indexed for SEO
  • The background-image property is fine for non-content (i.e. patterns, gradients), but is not accessible
  • The background-image offers sprite animation use cases (lacks "alt" attribute, so provide a text label)

Semantics and Accessibility

  • To make the web easier to use and access, and available to anyone with any device
  • Encompasses all disabilities, including visual, auditory, physical, speech, cognitive and neurological disabilities
  • Benefits people without disabilities as well
  • Accessible websites benefit from search engine optimisation (SEO)

Basic accessibility checklist (1/2)

  • Page title: To adequately and briefly describe the content of the page
  • Image text alternatives: To make visual information accessible
  • Headings: To provide meaningful hierarchy for facilitation of navigation
  • ARIA Landmark Tags: uses the landmark semantic tags
  • Contrast ratio: To have sufficient luminance contrast ratio, for people with different requirements - Web Tool
  • Resize text: To ensure visibility and usability as text size increases

Basic accessibility checklist (2/2)

  • Keyboard access & visual focus: To provide full functionality through a keyboard, and visible focus with logical order
  • Forms, labels & errors: To have proper labels, keyboard access, clear instructions, and effective error handling
  • Multimedia alternatives: To have alternative formats for audio and visual impaired (ARIA elements, transcripts)

HTML: Element Attributes

Configuring Tags

HTML Attributes

  • "Config Data" that modifies the element's behavior
  • They are place inside the opening tag
  • Multiple attributes are separated by a space
  • Have a name followed by an equal sign then a value.
  • The value should be wrapped in quotation marks (single or double, just be consistent)

Key/Value vs. Boolean Attributes

  • Most attributes are a "pair", they have a name (or key) and a value
  • Some attributes are Boolean attributes and don't need an explicit value
  • They will simply take on a value of true if they are present
