Node & The Terminal

Node.js

Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine that can run in a server environment.

Asynchronous nature means no dead-locking a serverside/database process.

The TSC (technical steering committee) ratifies changes that make it into each new version.

Download Node.js:

Running Node

Write a javascript file as you normally would.

// index.js
let myVar = 'Hello World!'
console.log(myVar)

Then in your terminal, execute the file with this command:

node index.js

Node Server

Node API Docs

const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});

Staging, Committing, Pushing & Pulling

Branching

Basic Git commands

  • git clone - copies a remote repo
  • git pull - pulls down latest changes
  • git add . - Stages all changes
  • git revert - unstages staged changes
  • git commit -m "Commit message" - Commits changes locally
  • git push - Pushes local commits to remote
  • git branch - displays branches
    • flags: -d <branchname> to delete a branch
  • git checkout <branchname> - Checks out branch
  • git checkout -b <branchname> - Creates & checks out new branch
  • git fetch - Fetches remote to view dif
  • git merge <branchname> - Merges in branch to current
  • git diff - Shows changes in red/green
  • git log - Shows the git history
    • flags:
      --graph --decorate --oneline
  • git reset --hard HEAD - Resets your local branch to the HEAD of remote
